FldHeader¶

class fld_header.FldHeader(nelgt, nx1, ny1, nz1, nelt=None, rdcode='', time=0.0, iostep=0, fid0=0, nfileoo=1, p0th=0.0, if_press_mesh=False, float_type=dtype('float32'), int_type=dtype('int32'), glel=None)[source]¶

Contains the header of a binary Nek5000 field file

The constructor is typically not used directly. Most users will prefer to use FldHeader.fromfile() or FldHeader.fromvalues().

Parameters

  • nelgt (int) – Number of global elements

  • nx1 (int) – Number of GLL gridpoints along x-axis

  • ny1 (int) – Number of GLL gridpoints along y-axis

  • nz1 (int) – Number of GLL gridpoints along z-axis

  • nelt (Optional[int]) – Number of elements in this file

  • rdcode (str) – String representing the fields contained in this file

  • time (float) – Absolute simulation time of this file’s state

  • iostep (int) – I/O timestep of this file’s state

  • fid0 (int) – Index of this file, with respect to all files produced at this I/O step

  • nfileoo (int) – Number of files produced at this I/O step

  • p0th (float) –

  • if_press_mesh (bool) – States whether pressure mesh is being used

  • float_type (dtype) – Data type used for floating point numbers in this file

  • int_type (dtype) – Data type used for integers in this file

  • glel (Optional[ndarray]) – Array of global element indices; shape must be (nelt,)
