FldData¶

class fld_data.FldData(header, coords=None, u=None, p=None, t=None, s=None)[source]¶

Contains the header and field data of a binary Nek5000 field file.

The constructor is typically not used directly. Most users will prefer to use FldData.fromfile() or FldData.fromvalues().

The parameters representing fields (coords, u, p, t, and s) must have sizes that are consistent with header. For example, the scalar field p must have size equal to (header.nelt * header.nx1 * header.ny1 * header.nz1,). The constructor will raise a ValueError if the sizes are inconsistent.

Parameters

  • header (FldHeader) – An instantiated fld_header.FldHeader object.

  • coords (Optional[ndarray]) – Array of global element coordinates. Must have shape = (header.ndims, header.nelt * header.nx1 * header.ny1 * header.nz1). Default is None.

  • u (Optional[ndarray]) – Array of velocities. Must have shape = (header.ndims, header.nelt * header.nx1 * header.ny1 * header.nz1). Default is None.

  • p (Optional[ndarray]) – Array of pressures. Must have shape = (header.nelt * header.nx1 * header.ny1 * header.nz1,). Default is None.

  • t (Optional[ndarray]) – Array of temperatures. Must have shape = (header.nelt * header.nx1 * header.ny1 * header.nz1,). Default is None.

  • s (Optional[ndarray]) – Array of passive scalars. Must have shape = (header.nscalars, header.nelt * header.nx1 * header.ny1 * header.nz1). Default is None.

Raises

ValueError – If the size of any field is inconsistent with the header.
